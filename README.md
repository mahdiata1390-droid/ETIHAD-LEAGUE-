# لیگ اتحاد (Ettehad League)

نسخه‌ی **کاملاً مسطح (flat)** — تمام فایل‌های HTML، CSS، جاوااسکریپت، آیکون‌ها، Manifest و Service Worker مستقیماً در همین یک پوشه قرار دارند؛ هیچ زیرپوشه‌ای وجود ندارد. این ساختار مخصوص آپلود راحت از گوشی به گیت‌هاب طراحی شده است.

بدون نیاز به Node.js، npm، یا هیچ مرحله‌ی Build ای — فقط فایل‌های استاتیک + Firebase.

## فهرست فایل‌ها

| فایل | نقش |
|---|---|
| `index.html` | صفحه‌ی اصلی و تنها فایل HTML اپ (SPA) |
| `style.css` | تمام استایل‌ها (تم شیشه‌ای مشکی/طلایی/آبی، راست‌چین) |
| `manifest.json` | تنظیمات PWA برای نصب روی گوشی |
| `service-worker.js` | کش‌کردن فایل‌ها برای کارکرد آفلاین |
| `offline.html` | صفحه‌ی نمایش‌داده‌شده وقتی اینترنت قطع است |
| `icon-192.png` `icon-512.png` `icon-512-maskable.png` `icon-apple-touch.png` | آیکون‌های PWA |
| `firebase-config.js` | اتصال به Firebase (کلیدها را اینجا وارد کنید) |
| `auth-state.js` | مدیریت ورود مدیر/بازیکن |
| `access-guard.js` | محافظت از صفحات مدیر/بازیکن |
| `firestore-service.js` | تمام عملیات خواندن/نوشتن در Firestore و Storage |
| `standings.js` | محاسبه‌ی خودکار جدول لیگ و گلزنان از روی نتایج |
| `date-utils.js` | تبدیل تاریخ میلادی به شمسی |
| `router.js` | مسیریابی مبتنی بر هش (بدون نیاز به تنظیم سرور) |
| `ui-helpers.js` | توابع کمکی رندر (لوگو، بج، اسپینر و...) |
| `component-*.js` | تکه‌کدهای قابل‌استفاده‌ی مجدد (کارت تیم، کارت مسابقه، فرم ثبت نتیجه و...) |
| `page-*.js` | هر فایل یک صفحه‌ی کامل از اپ (خانه، جدول، تیم‌ها، پنل مدیر و...) |
| `nav-footer.js` | نوار بالا و پایین سایت |
| `app.js` | نقطه‌ی شروع؛ همه‌چیز را به هم وصل می‌کند |
| `firestore.rules` / `storage.rules` | قوانین امنیتی Firebase |

## گام ۱: ساخت پروژه‌ی Firebase

۱. به [Firebase Console](https://console.firebase.google.com) بروید و پروژه‌ای بسازید.
۲. **Build > Authentication** → «Get started» → روش **Email/Password** را فعال کنید.
۳. در تب **Users**، یک حساب برای خودتان (مدیر لیگ) بسازید و مقدار **UID** آن را کپی کنید.
۴. **Build > Firestore Database** → یک دیتابیس بسازید (Production mode).
۵. **Build > Storage** → یک باکت بسازید (Production mode).
۶. **Project settings ⚙️ > General** → در «Your apps» یک اپ وب (</>) اضافه کنید و مقادیر `firebaseConfig` را کپی کنید.

## گام ۲: وارد کردن کلیدهای Firebase در کد

فایل `firebase-config.js` را باز کنید و مقادیر را جایگزین کنید:

```js
export const firebaseConfig = {
  apiKey: "...",
  authDomain: "...",
  projectId: "...",
  storageBucket: "...",
  messagingSenderId: "...",
  appId: "..."
};

export const ADMIN_UID = "همان UID مدیر از گام ۱";
```

> **نکته:** برخلاف کلید API سرویس‌های دیگر، `firebaseConfig` وب یک مقدار عمومی و غیرمحرمانه است — امنیت واقعی توسط `firestore.rules` و `storage.rules` تأمین می‌شود، نه با پنهان‌کردن این مقادیر. پس مشکلی ندارد که این فایل مستقیماً در گیت‌هاب باشد.

## گام ۳: اعمال قوانین امنیتی

در `firestore.rules` و `storage.rules`، عبارت `REPLACE_WITH_ADMIN_UID` را با همان UID مدیر جایگزین کنید.

سپس (از کامپیوتر، یک‌بار) با Firebase CLI اعمال کنید:

```bash
npm install -g firebase-tools
firebase login
firebase deploy --only firestore:rules,storage:rules
```

اگر به کامپیوتر دسترسی ندارید، می‌توانید همین متن را مستقیم در Firebase Console بچسبانید: **Firestore Database > Rules** و **Storage > Rules**، سپس Publish را بزنید.

## گام ۴: آپلود به گیت‌هاب از گوشی

۱. در اپ گیت‌هاب یا مرورگر گوشی، یک ریپازیتوری جدید بسازید.
۲. تمام فایل‌های این پوشه را (بدون هیچ زیرپوشه‌ای) در ریشه‌ی ریپازیتوری آپلود کنید.
۳. Commit کنید.

## گام ۵: انتشار (Hosting رایگان)

چون هیچ Build ای لازم نیست، هر سرویس Hosting استاتیکی کار می‌کند:

- **Vercel:** ریپازیتوری را Import کنید؛ چون فریمورکی تشخیص داده نمی‌شود، Vercel آن را به‌عنوان استاتیک منتشر می‌کند (نیازی به تنظیم دستور Build نیست).
- **GitHub Pages:** از تنظیمات ریپازیتوری، Pages را از شاخه‌ی اصلی (root) فعال کنید.
- **Netlify / Firebase Hosting:** ریپازیتوری را وصل کنید؛ پوشه‌ی publish را روی ریشه (`/`) بگذارید.

بعد از انتشار، آدرس دامنه را در Firebase Console از **Authentication > Settings > Authorized domains** اضافه کنید تا ورود کار کند.

## گام ۶: افزودن حساب برای بازیکنان

چون Firebase از سمت کلاینت اجازه نمی‌دهد برای شخص دیگری حساب بسازید:

۱. در Firebase Console از **Authentication > Users > Add user** یک حساب برای بازیکن بسازید.
۲. UID آن را کپی کنید.
۳. در **پنل مدیر > مدیریت بازیکنان**، پایین کارت همان بازیکن، UID را وارد کرده و «اتصال» را بزنید.

بازیکن از این پس با همان ایمیل/رمز وارد می‌شود و فقط اطلاعات و آمار خودش را در «پنل بازیکن» می‌بیند.

## گام ۷: نصب PWA روی گوشی

- **اندروید (Chrome):** منوی سه‌نقطه → «Add to Home screen»
- **آیفون (Safari):** دکمه‌ی Share → «Add to Home Screen»
- **کامپیوتر (Chrome/Edge):** آیکون نصب کنار نوار آدرس

## اجرای محلی (اختیاری، برای تست قبل از آپلود)

چون فایل‌ها از نوع ES Module هستند، مرورگر برای اجرای صحیح به یک سرور محلی ساده نیاز دارد (باز کردن مستقیم `index.html` با `file://` کار نمی‌کند):

```bash
npx serve .
# یا
python3 -m http.server 8000
```

سپس آدرس چاپ‌شده را در مرورگر باز کنید.

## نکات امنیتی
- اسم/ایمیل مدیر هیچ‌جای رابط کاربری نمایش داده نمی‌شود؛ فقط UID او (یک شناسه‌ی فنی، نه اطلاعات شخصی) در `firebase-config.js` و قوانین امنیتی قرار می‌گیرد.
- تمام نوشتن‌های حساس (تیم/بازیکن/مسابقه/خبر) هم در کد و هم در `firestore.rules` محدود به همان UID است؛ حتی دورزدن رابط کاربری اجازه‌ی نوشتن نمی‌دهد.
- بازیکنان فقط اجازه‌ی خواندن سند اتصال حساب خودشان را دارند و هیچ مسیر نوشتنی برایشان باز نیست.
